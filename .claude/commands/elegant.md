---
description: Challenge the current implementation for a more elegant solution. Use before finalizing non-trivial changes.
---

Pause and evaluate the current implementation:

1. Is this the most elegant solution given everything known now?
2. Does any part feel like a workaround? If yes — what is the real fix?
3. Would removing or simplifying anything make this clearer?
4. Is there a single abstraction that eliminates current complexity?

If a better approach exists: describe it, then implement it.
If the current approach is clean: say so and explain why.

Do not over-engineer. Skip for simple, obvious changes.
